Роботизированный обзвон пациентов с целью подтверждения записи

Реализовано:

  • С помощью голосового помощника с функцией распознования речи запущен опрос о статусе записи к специалисту (записаться, отменить, перенести).
  • Подключена переадресация на администратора медицинской клиники для решения вопросов клиента.
  • Помощник при разговоре обращается к клиенту по ФИО и все полученные данные заносит в систему клиники.

Результат:

  • Благодаря витруальному сотруднику, сократились расходы на обзвон пациентов.
  • Сотрудник разгружен, только при необходимости решать вопрос клиента
  • Помощник при разговоре обращается к клиенту по ФИО и все полученные данные заносит в систему клиники.

Интеграция по HTTP API для обзвона клиентов

Реализовано:

  • Организация API c использованием уникальных переменных, с номером телефона клиента/абонента и привязкой ID номера офиса
  • Организованна система переадресации звонка, при успешном ответе абонента, в офис продаж с учётом места проживания абонента
  • Благодаря автоматически масштабируемой системы были достигнуты максимальные показатели по обработке входящих заявок (выросло до 350 000 запросов в день).
  • Разработана и внедрена система для самостоятельного изменения логики сценария звонка, со стороны самого клиента.
  • Запущена система по обработке номера абонента с учётом UTC и местонахождения
  • Постоена логика CallBack (статус звонка) по каждому звонку
  • Предоставлен индивидуальный доступ к статистике

Результат:

  • Снизилась нагрузка на операторов офисов продаж
  • Благодаря автоматизированной системе UNIBELL увеличилась конверсия выданных займов на 4%
  • Под каждый офис продаж заказчика были заведены "компании" для обзвона, благодаря этому был достигнут контроль качества по каждому звонку и в интерфейсе платформы отображался каждый звонок.
